Connect your Supabase database

This system uses Supabase (free) as its back end — a hosted database that keeps all data live and accessible from any device. You only need to do this setup once.

How to set this up (takes ~10 minutes)
1
Go to supabase.com → create a free account → New Project
2
In your project: Settings → API → copy your Project URL and anon/public key into the fields above
3
In Supabase: go to SQL Editor → paste and run the setup SQL shown inside the app after connecting
4
Create user accounts in Supabase Authentication → Users — set role to admin or caller in user metadata

Database setup
Run this SQL in your Supabase SQL Editor to create all required tables
Step 1 — Run setup SQL in Supabase
Go to your Supabase project → SQL Editor → paste and run the SQL below.
-- Simtec Therapeutic — Database Setup SQL -- Run this once in Supabase SQL Editor CREATE TABLE IF NOT EXISTS orders ( id uuid DEFAULT gen_random_uuid() PRIMARY KEY, created_at timestamptz DEFAULT now(), fname text NOT NULL, lname text NOT NULL, phone text, email text, address text, consultant text, consultant_phone text, pay_day text, pay_type text DEFAULT 'dd', total numeric, weekly_rep numeric, items jsonb, call_status text DEFAULT 'pending', -- pending | complete call_notes text, call_time timestamptz ); -- Enable real-time for instant queue updates ALTER TABLE orders REPLICA IDENTITY FULL; ALTER PUBLICATION supabase_realtime ADD TABLE orders; -- Row level security (open for now — tighten later) ALTER TABLE orders ENABLE ROW LEVEL SECURITY; CREATE POLICY "Allow all authenticated" ON orders FOR ALL USING (auth.role() = 'authenticated') WITH CHECK (auth.role() = 'authenticated');
Step 2 — Create user accounts
In Supabase → Authentication → Users → Invite user. After creating, edit the user's raw metadata to set their role:
-- In user metadata (raw_user_meta_data), set: { "role": "admin" } -- full system access { "role": "caller" } -- confirmation calls only
Step 3 — Share the URL
Once set up, share this HTML file (or host it) with any staff member. They log in with their Supabase credentials. Admins see everything. Callers see only their queue.
